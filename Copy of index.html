Ideia

  1. Implementar o "javascript unobstrusivo".
  2. Diminuir as implementações em javascript.
  3. Acessivel à "não programadores" (designer na maioria dos casos).
  4. Focar na idéia de que a página, os conteúdos, possam funcionar sem o javascript.
  5. Aproveitar a "semântica" das tags para implementar componentes.
  6. O html (view) não deve conhecer o javascript

Implementação

  1. Explorar a ancora (#) para que ações javascript sejam executadas. Usar como base o conceito das rotas mvc do .net e outros frameworks.
  2. Facilidade de se criar componentes e/ou modulos
  3. Vinculado diretamente ao jquery
  4. Gerar objeto json a partir de campos de um formulário.
  5. Os componentes/modulos fiquem descoplados;
  6. Evitar que se torne uma "linguagem" de programação;

Controladores

Todo o desenvolvimento é baseado em pequenos componentes. O core representa alguns componentes que irão executar os outros componentes registrados.

Unobstrusive

Ativa módulos que estão viculados aos elementos do html.

  • Momento que será ativo estes módulos:
    1. Ao carregar a página
    2. Quando adicionar conteúdo html na página atravez das seguintes comando do jquery:
      • html
      * este métodos foram reescritos para que todo conteúdo novo que entra na página passa executar e controlador
  • Registrando um modulo a este controlador:
    1. Criar um componente seguindo o padrão comum de componentes.
    2. O componente deverá ter o metodo render que irá receber o elemento jquery.
    3. No html incluir o parametro data-type que irá receber como valor o name do componente
